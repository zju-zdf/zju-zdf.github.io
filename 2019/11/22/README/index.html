产品经理能力矩阵

​ 业内对于产品经理能力模型的归纳屡见不鲜，而我将结合日常工作，从个人理解的层面，以作品demo的形式展开（详细文档见各标题对应文件夹），总结为如下13个方面的产品硬性能力。

1.战略规划定位

​ 一款好的产品是有灵魂的，优美的设计、技术、运营都能体现背后的理念，是能主动吸引用户来使用的。对于产品经理而言，正确把控公司战略，全面解读公司规划，做好火车头的工作，选择往往比努力更重要。

​ 附件文档讲述了普惠金融的由来与发展，产品体系的构建思路与现状分析，三产品详细介绍与发展策略，最终总结规划今后工作重心。

2.架构设计规划

​ 架构域包括：业务架构、数据架构、应用架构、技术架构等。业务架构是战略，应用架构是承上启下，一方面承接业务架构的落地，另一方面影响技术架构的选型，而数据是一家企业最宝贵的资产。

​ 附件文档讲述了业务架构规划的背景、分析及方案思路，并从系统架构角度开展分层服务架构建设和IT基础架构提升，构建信贷六大支柱能力。

3.产品需求规划

​ 产品路线图是一个计划纲领，提供了一系列与产品战略相一致的战术步骤，将产品推向预定方向的轨道，是整个团队一起努力的方向指引。

​ 附件文档描述了各阶段产品规划所明确的主题目标，需求优先级安排，及需求的分析和规模预估。

4.产品原型设计

​ 产品UI设计原则：便捷性，美观性，容错性，易学性，情感化。视觉把控角度：布局、色彩、字体、图标。

​ 附件文档是用Axure绘制的高保真原型，更多设计作品见链接：axshare。

5.产品业务流程

重点打造信用类经营快贷、抵质押类e抵快贷和产业链e链快贷三大产品体系，满足小微企业不同场景融资需求。

  • 经营快贷：运用大数据及互联网技术，根据不同场景，基于客户的交易、资产、纳税等多维度数据构建客户准入、额度测算、风险监测模型，为小微企业主提供在线信用类融资服务。
  • e抵快贷：针对持有优质房产的小微客户提供的在线抵押经营贷款产品。
  • e链快贷：以核心企业为依托，应用区块链和大数据技术，打通供应链全产业的金融服务需求。

6.接口服务体系

​ 构建以共享服务中心为核心的服务分层体系，形成“横向能解耦、纵向有层次”的灵活IT架构。通过应用系统之间的解耦和组合，以统一的方式对外提供服务接口，从而实现应用系统之间的共享和协作。

​ 附件文档体现了小微e贷三产品的服务分层体系，以及对外封装的API接口，逐步完善普惠金融API生态圈。

7.商业需求文档

​ 基于商业目标或价值所描述的产品需求内容报告， 包括成本，收益，商业，市场多方面 ，通常是供决策层们讨论的演示文档。

​ 附件文档以烟草贷为案例，简要介绍了产品定义、市场分析、商业合作、路线规划等。

8.产品需求文档

​ 使产品从“概念化”阶段进入到“图纸化”阶段的主要文档，旨在对项目的业务架构、产品流程、功能需求做详细的介绍，为产品后续的需求、设计、开发、测试、上线提供准确指导和依据。

​ 附件文档讲述了业务背景、用户分析、项目必要性、总体思路、原型交互、业务流程及相关细化方案等。

9.用户调研报告

​ 其目的是通过用户研究，理解用户行为和想法，得出有利于优化产品性能、提升产品体验的结论，最终达到提升产品综合竞争力的目的。

​ 附件文档讲述了普惠金融服务分行调研的目的、对象、方法、内容和总结分析。

10.竞品分析报告

​ 明确市场趋势和业界现状，竞争对手产品定位和发展策略，关注其核心功能、交互设计、产品优缺点。

​ 附件文档通过选取中行供应链，与其比较发展策略、产品体系、管理模式、技术制度，并作总结分析。

11.运营分析报告

​ 无数据不决策，通过不停的数据挖掘试验，消除一定程度对未来的不确定性。借鉴AARRR模型，从产品生命周期各阶段解读关键指标，为后续工作开展提供指引。

​ 附件文档分析了普惠金融产品2019上半年的运营报告，其中关键数据已脱敏。

12.关键指标体系

​ 核心指标对于产品运营来说至关重要，它是你的产品提供给用户最本质和最为核心的操作，也是最能体现用户参与度的操作指标。

​ 附件文档总结了普惠金融领域六个方面的关键指标体系：金额、利息、客户、渠道、特色、其它。

13.精益研发管理

​ 聚焦用户价值，提升研发效能，加速需求价值流动，解决效率竖井问题。

​ 附件文档总结了四个方面的管理思路：需求、开发、测试、投产。
